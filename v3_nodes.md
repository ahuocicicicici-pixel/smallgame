## 核心机制 · cost + beat 双字段

每个 choice 数据结构:

```js
{
  key: "A",
  text: "选项主文本(玩家选前看到)",
  cost: "你即将放弃什么 · 1-2 句话",         // 锈橘/暗黄色 · 静态预告
  beat: "选完后弹的剧情段落",                  // 浮窗演出 · 必须按确认
  effects: { 决: 1, 共: 3, 规: -1, 险: 1 },
  next: "N02",
  flag: "prioritized_lu",
  // 可选:item / condition
}
```

**渲染**:
- `cost` 在选项卡片末尾以【代价 · ...】块出现 · 锈橘色 + 左竖线 + 半透明底
- `beat` 选完后弹 `.consequence-beat` 浮窗 · 居中 · 必须按"知道了"才进 next 节点

**为什么这样设计**:
- v22 / v2.2 的代价是隐藏数值变化 → 玩家感受不到选择的重量
- 单 cost 字段 → 玩家选前知道代价,但选完后没有"代价兑现"的演出
- cost + beat → 选前预告 + 选后兑现,把每个选择变成**两次注视**:一次理性掂量,一次情感落地
